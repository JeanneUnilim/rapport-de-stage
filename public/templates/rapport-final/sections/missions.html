01 — Description du projet et missions réalisées

Un back-office pour le
Centre Initiation Nature

Le projet

Ma mission principale a consisté à concevoir et développer une application web de back-office pour le Centre Initiation Nature (CIN) de Laval Agglomération. Ce back-office permet à l'équipe administrative du CIN de gérer l'intégralité de sa programmation d'activités et des inscriptions associées depuis un espace centralisé et sécurisé.

Le périmètre couvre la gestion des activités, des inscriptions, des catégories, des animateurs, l'envoi automatique d'e-mails, l'export CSV et un agenda interactif. Le front-office public destiné aux citoyens est prévu dans un second temps, hors périmètre de ce stage, vers Noël 2026.

La stack technique

Le projet repose sur une architecture découplée : la partie visible (le front-end) et la partie qui gère les données (le back-end) sont deux applications distinctes qui communiquent via des requêtes HTTP (le front-end envoie une demande, le back-end répond avec des données, sans que l'un n'ait accès direct au code de l'autre).

Côté front-end : Angular 21 pour la logique de l'interface, PrimeNG 20 pour les composants visuels, Tailwind CSS pour la mise en page et FullCalendar pour l'agenda interactif.

Côté back-end : Node.js avec Express 5 pour l'API REST, MongoDB via Mongoose pour la base de données, et Nodemailer pour les e-mails. Versionné sur GitLab avec des Conventional Commits (une convention de nommage des commits pour structurer l'historique : feat:, fix:, refactor:…).

Structure du projet

Les deux applications sont totalement indépendantes : Angular ne sait pas comment MongoDB stocke les données, et Node.js ne sait pas comment Angular affiche l'interface. Elles communiquent uniquement via des requêtes HTTP : le front-end envoie une requête, le back-end répond avec des données JSON.

Côté front-end, Angular organise le code en composants. Chaque composant représente un morceau d'interface (un formulaire, une table, un dialog...) et est composé de 4 fichiers : le .ts (TypeScript) contient la logique, le .html définit la structure visuelle, le .scss gère les styles spécifiques, et le .spec.ts est prévu pour les tests unitaires.

Côté back-end, le code suit une architecture MVC (Modèle-Vue-Contrôleur), un modèle classique qui sépare les responsabilités en trois couches : les models décrivent la structure des données en base, les controllers contiennent la logique métier (que faire quand une requête arrive), et les routes définissent les URLs de l'API qui font le lien entre les deux. Les services viennent en complément pour les tâches transversales comme l'envoi de mails ou les exports.
